npm安装提示：checkPermissions

发表于 2018-05-10 | 分类于 工具

操作步骤是这样的：在用到axios时，就npm install --save axios，安装完了，就开始调试代码：npm run dev，然后就缺失组件啥的。

我用git diff查看代码改动了哪些，结果显示没什么问题，改动不多。就想着可能是node_modules依赖包的问题，接着重新安装一下应该没问题。

不管是淘宝的源，还是淘宝的源，都会显示：checkPermissions Missing write access反正就是安装不了。网上搜索了下，说权限问题，要安装sudo。我的系统是Win10，之前也没有遇到这个问题，我也不想安装乱七八糟的东西，毕竟Windows的尿性在那。

也有人说要删除node_modules安装包，然后重新安装下。然后我尝试了下，果然解决了。

其实拿Win10开发前端，有点不太合适。首先，显示问题：我的电脑是1366 X 768，分辨率挺渣的。其次涉及到后端的东西，Windows安装依赖包，还是挺渣的。

曾经捣腾过黑苹果，自己电脑的渣分辨率和黑苹果的驱动，折磨的不要不要的，真不想搞了。也搞过双系统（Windows+Ubuntu），每次删除某个分区或者新建分区，Ubuntu的启动就废了，又得修复引导，太累。也搞过虚拟机VMWare，那速度挺感人的。现在，换成三星250G固态硬盘，确实比西数128G的硬盘快很多，当然，虚拟机里运行webpack，还是很慢的。编译时，就不能使用其它软件，磁盘活动时间也会达到100%，唯一的好处就是随时可以把虚拟机弄到其它地方。后续准备体验下Docker，这样以后维护服务器会方便很多，此项工作在计划中……

家里有台MacBook Air，由于轻薄，被老婆用来办公和出差，谁让我心疼老婆呢！
